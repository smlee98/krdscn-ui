## 작동 방식

krdscn/ui는 저장소를 복제해서 사용하는 패키지가 아니라, CLI에 전달하는 원격 레지스트리입니다.

```bash
pnpm dlx shadcn@latest add https://krdscn.gwlee.kr/r/krds-all.json
```

CLI는 레지스트리 URL을 읽고 필요한 의존성, 유틸리티, 훅, 컴포넌트 소스 코드를 현재 프로젝트에 추가합니다. 설치 이후의 컴포넌트는 여러분의 코드베이스에 속하므로 서비스 요구사항에 맞게 자유롭게 수정할 수 있습니다.

## 제공 항목

| 항목 | 설명 |
| --- | --- |
| `krds-utils` | 컴포넌트에서 사용하는 `cn` 유틸리티 |
| `use-mobile` | 반응형 컴포넌트를 위한 모바일 감지 훅 |
| `krds-all` | 전체 UI 컴포넌트 묶음 |
| `button`, `dialog`, ... | 필요한 컴포넌트만 설치하기 위한 개별 항목 |

## 호환성 전략

기존 Radix primitive 생태계와 자연스럽게 맞물리도록 컴포넌트 구조를 유지합니다.

- icon library: `lucide`

## 업데이트 방식

krdscn/ui는 소스 코드를 프로젝트에 복사하는 오픈 코드 모델을 따릅니다. 업데이트가 필요하면 같은 레지스트리 URL로 다시 설치하고, CLI가 보여주는 변경 사항을 확인한 뒤 프로젝트에 맞게 반영하세요.
