컴포넌트 하나하나의 API는 [컴포넌트 문서](/docs/components)에 있습니다. 가이드는 그 반대편에서 시작합니다 — **"공공 서비스 화면을 만들어야 하는데, 뭘 어떤 순서로 조합하지?"**

각 가이드는 하나의 시나리오를 처음부터 끝까지 따라 하는 형식입니다. 순서대로 읽으면 빈 프로젝트에서 완성된 화면까지 이어지고, 필요한 것만 골라 읽어도 됩니다.

## 목차

1. **[시작하기](/docs/guides/getting-started)** — 빈 Next.js 프로젝트에 첫 KRDS 버튼을 띄우기까지
2. **[페이지 셸 만들기](/docs/guides/page-shell)** — 공식 배너·헤더·푸터로 공공 사이트 기본 골격 잡기
3. **[신청 폼 만들기](/docs/guides/building-a-form)** — 입력·선택·날짜·검증까지, 접근성이 따라오는 폼
4. **[내비게이션 구성하기](/docs/guides/navigation)** — 메인 메뉴·브레드크럼·사이드 메뉴·페이지네이션
5. **[목록과 표 페이지](/docs/guides/data-pages)** — 표·구조화 목록·배지로 데이터 화면 만들기
6. **[다크 모드 켜기](/docs/guides/dark-mode)** — KRDS 고대비 모드를 클래스 하나로
