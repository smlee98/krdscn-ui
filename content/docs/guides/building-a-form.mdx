title: 신청 폼 만들기
description: 입력·선택·날짜·동의·검증까지, 접근성이 자동으로 따라오는 KRDS 신청 폼을 만들어봅니다.
---

공공 서비스의 절반은 폼입니다. 이 가이드에서는 "프로그램 신청" 폼을 만들면서 krdscn/ui 폼 컴포넌트의 핵심 패턴 — 특히 **`error` 프롭 하나로 접근성까지 처리되는 방식** — 을 익힙니다.

## 설치

```bash
pnpm dlx shadcn@latest add \
  https://krdscn-ui.smlee.info/r/text-input.json \
  https://krdscn-ui.smlee.info/r/select.json \
  https://krdscn-ui.smlee.info/r/checkbox.json \
  https://krdscn-ui.smlee.info/r/date-input.json \
  https://krdscn-ui.smlee.info/r/button.json
```

<Steps>

<Step>

### 텍스트 입력 — 레이블과 도움말

```tsx
import { TextInput } from "@/components/ui/krds/(input)/text-input"

;<TextInput label="이름" placeholder="홍길동" hint="주민등록상 이름을 입력하세요" />
```

`label`을 넘기면 `<label>`과 입력 요소가 자동으로 연결됩니다. `hint`는 입력 아래 도움말로 렌더되고, 입력 요소의 `aria-describedby`에 연결됩니다.

</Step>

<Step>

### 에러는 프롭 하나로

검증 실패 시 `error`에 메시지를 넘기기만 하면 됩니다.

```tsx
<TextInput label="이메일" placeholder="you@example.com" error="이메일 형식이 아닙니다" />
```

이 한 줄이 하는 일:

- 입력 아래에 에러 메시지 렌더 (KRDS 규칙: 항상 컨트롤 아래)
- 메시지에 `role="alert"` — 스크린리더가 즉시 낭독
- 입력 요소에 `aria-invalid="true"`와 `aria-describedby` 자동 연결
- 테두리가 danger 색으로 — 시각 스타일이 `aria-invalid` 셀렉터에 걸려 있어서, 화면과 보조기술이 어긋날 수 없습니다

`success`, `information` 프롭도 같은 방식으로 동작합니다 (우선순위: error > success > information > hint).

</Step>

<Step>

### 선택 — Select와 Checkbox

Select는 `options` 배열을 받습니다.

```tsx
import { Select } from "@/components/ui/krds/(selection)/select"
import { Checkbox } from "@/components/ui/krds/(selection)/checkbox"

const PROGRAM_OPTIONS = [
  { value: "tour", label: "견학 프로그램" },
  { value: "edu", label: "교육 프로그램" },
  { value: "event", label: "체험 행사" },
]

<Select options={PROGRAM_OPTIONS} label="신청 프로그램" />
<Checkbox label="개인정보 수집·이용에 동의합니다" />
```

둘 다 Radix Primitives 기반이라 키보드 인터랙션(방향키 탐색, Space 토글)과 ARIA 속성이 기본 내장입니다.

</Step>

<Step>

### 날짜 — DateInput

```tsx
import { DateInput } from "@/components/ui/krds/(input)/date-input"

;<DateInput label="방문 희망일" />
```

트리거를 누르면 달력 팝오버가 열립니다. 직접 타이핑은 의도적으로 막혀 있어서 — 값은 달력에서만 선택됩니다 — 자유 입력의 형식 오류를 원천 차단합니다.

</Step>

<Step>

### 조합 — 신청 폼 완성

```tsx title="app/apply/page.tsx"
"use client"

import { useState } from "react"
import { Button } from "@/components/ui/krds/(action)/button"
import { TextInput } from "@/components/ui/krds/(input)/text-input"
import { DateInput } from "@/components/ui/krds/(input)/date-input"
import { Select } from "@/components/ui/krds/(selection)/select"
import { Checkbox } from "@/components/ui/krds/(selection)/checkbox"

const PROGRAM_OPTIONS = [
  { value: "tour", label: "견학 프로그램" },
  { value: "edu", label: "교육 프로그램" },
]

export default function ApplyPage() {
  const [email, setEmail] = useState("")
  const [submitted, setSubmitted] = useState(false)

  const emailError = submitted && !email.includes("@") ? "이메일 형식이 아닙니다" : undefined

  return (
    <form
      className="mx-auto flex max-w-[480px] flex-col gap-6 py-10"
      onSubmit={(e) => {
        e.preventDefault()
        setSubmitted(true)
      }}
    >
      <TextInput label="이름" placeholder="홍길동" />
      <TextInput
        label="이메일"
        placeholder="you@example.com"
        value={email}
        onChange={(e) => setEmail(e.target.value)}
        error={emailError}
      />
      <Select options={PROGRAM_OPTIONS} label="신청 프로그램" />
      <DateInput label="방문 희망일" />
      <Checkbox label="개인정보 수집·이용에 동의합니다" />

      <Button type="submit" size="lg">
        신청하기
      </Button>
    </form>
  )
}
```

</Step>
