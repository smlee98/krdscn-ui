title: 다크 모드 켜기
description: KRDS 고대비 모드를 클래스 하나로 켜는 방법과, 색이 새지 않게 지키는 규칙을 다룹니다.
---

krdscn/ui의 다크 모드는 별도 설정 파일이 없습니다. 컴포넌트를 하나라도 설치했다면 토큰 스타일시트(`krds-theme` → `app/krds.css`)가 이미 들어와 있고, 그 안에 다크 모드가 준비돼 있습니다. **`<html>`에 `.dark` 클래스를 토글하는 것**이 여러분이 할 일의 전부입니다.

## 원리 — 시맨틱 토큰만 바뀐다

KRDS는 다크 모드를 "고대비 모드"로 정의합니다. `krds.css`는 원시 색상 팔레트(프리미티브)는 그대로 두고, 역할 토큰(시맨틱)만 `.dark`에서 리매핑합니다.

```css title="app/krds.css (발췌)"
:root {
  --krds-foreground-primary: var(--krds-color-primary-60);
  --krds-surface: var(--krds-color-gray-0);
}

.dark {
  --krds-foreground-primary: var(--krds-color-primary-20);
  --krds-surface: var(--krds-color-gray-100);
}
```

컴포넌트는 `bg-krds-surface`, `text-krds-foreground-primary` 같은 시맨틱 클래스만 쓰기 때문에, 클래스 하나로 전체 색이 일관되게 뒤집힙니다.

<Steps>

<Step>

### next-themes 설정

```bash
pnpm add next-themes
```

`attribute="class"`가 핵심입니다 — next-themes가 `<html>`에 `.dark`를 넣고 빼줍니다.

```tsx title="app/layout.tsx"
import { ThemeProvider } from "next-themes"

export default function RootLayout({ children }) {
  return (
    <html lang="ko" suppressHydrationWarning>
      <body>
        <ThemeProvider attribute="class" defaultTheme="system" enableSystem>
          {children}
        </ThemeProvider>
      </body>
    </html>
  )
}
```

`suppressHydrationWarning`은 next-themes가 서버 렌더 후 클래스를 조정할 때 나는 하이드레이션 경고를 막아줍니다.

</Step>

<Step>

### 토글 버튼

```tsx title="components/mode-toggle.tsx"
"use client"

import { useTheme } from "next-themes"
import { Button } from "@/components/ui/krds/(action)/button"

export function ModeToggle() {
  const { resolvedTheme, setTheme } = useTheme()

  return (
    <Button variant="text" size="sm" onClick={() => setTheme(resolvedTheme === "dark" ? "light" : "dark")}>
      {resolvedTheme === "dark" ? "라이트 모드" : "다크 모드"}
    </Button>
  )
}
```

이게 전부입니다. krdscn-ui 문서 사이트의 우측 상단 토글도 정확히 이 방식으로 동작합니다.

</Step>

</Steps>

## 색이 새지 않게 지키는 규칙

다크 모드가 "거의" 되는데 몇 군데만 색이 안 바뀐다면, 십중팔구 하드코딩입니다. 직접 작성하는 코드에서 두 가지만 지키면 됩니다.

- **hex와 고정 색 클래스를 쓰지 마세요.** `bg-white`, `text-[#333]`은 테마를 무시합니다. `bg-krds-surface`, `text-krds-foreground`처럼 시맨틱 토큰 클래스를 쓰면 자동으로 전환됩니다.
- **프리미티브보다 시맨틱을 우선하세요.** `bg-krds-gray-0`도 다크에서 안 바뀝니다(프리미티브는 고정값). "이 자리의 역할"에 맞는 시맨틱 토큰(`surface`, `foreground`, `border` 계열)을 고르는 게 원칙입니다.

<Callout>
  krdscn/ui 저장소는 이런 색 누수를 잡기 위해 라이트/다크 렌더링을 픽셀 단위로 비교하는 검증 스크립트를 돌립니다.
  여러분의 프로젝트에서도 의심되는 화면은 두 테마로 나란히 띄워보는 것만으로 대부분 잡힙니다.
</Callout>
